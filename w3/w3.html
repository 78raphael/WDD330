Chapter 5: Objects

Notes

  • Objects are a set of related self contained group of functions and values.
  • Objects are used to keep related values and methods together in the same place.
  • An object literal is wrapped in curly braces: {}. A useful way to organize code into a single unit.
  • Objects are mutable, meaning properties and methods are able to be changed or removed
  • Use of dot notation is a way of accessing properties in an object. EX. object.propertyName
  • Bracket notation is also a way to access properties. EX.object['propertyName']


  • delete operator can remove a property from an object: EX. delete object.propertyName
  • keyword {{this}} refers to the object that is the property or method is within
  • Namespacing: crating an object literal that serves as a distict object for properties and values. One advantage this has is if multiple methods use the same name, but need differnt objects to work within


JSON:

  1. Property names must be double-quoted
  2. Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects
  3. Functions are not permitted values
  • parse(): takes a string of data in JSON format and returns a JavaScript object
  • stringify(): takes a JavaScript object and returns a string of JSON data


  • The Epoch is a time that starts on 1 January 1970, and counts using seconds from that date
  • UTC is much the same as Greenwich Mean Time.


Regular Expressions

  • greedy, which means they will match the longest possible string
  • lazy modifiers that match the shortest possible string by adding an extra '?' after the modifier.
  • Sub-patterns can be created inside a regular expression by placing them inside parentheses. These are known as capturing groups

Chapter 6: Document Object Model

  • The Document Object Model, or DOM for short, represents an HTML document as a network of connected nodes that form a tree-like structure
  • The DOM treats everything on a web page as a node
  • Document.body returns the body element of a web page, as we saw in the previous example.
  • Document.images returns a node list of all the images contained in the document.
  • Document.links returns a node list of all the elements and elements that have an href attribute.
  • Document.anchors returns a node list of all the elements that have a name attribute.
  • Document.forms returns a node list of all the forms in the document

  • The document.querySelector() method allows you to use CSS notation to find the first element in the document that matches that matches a CSS selector provided as an argument
  • The childNodes property is a list of all the nodes that are children of the node concerned
  • The node lists returned by the document.getElementsByClassName() and document.getElementsByTagName() methods are 'live collections'

Chapter 7: Events

  • Events provides the link between the web page and user interactions
  • Clicking on a link, pressing a key, or moving a mouse, an event occurs that our program can detect and then respond to

  • Event listeners allow the the program to respond when the users responds to the website in certain, specific ways. These are actions taken by the program to respond when actions occur on the page, such as clicking on certain buttons or adding text to a box.

There are differnt ways to invoke event handlers:

  1. Inline Event Handlers: adding event handlers directly into the HTML.
  2. Use Event Handler properties in JavaScript to assign a function.
  3. Add Event Listners, this allows multiple functions to be attached independently to different events: addEventListener()

  • The type property returns the type of event that occurred.
  • The target property returns a reference to the node that fired the event.
    1. Mouse events
    2. Keyboard events
    3. Pressing Modifier keys, such as Shift, Ctrl, Alt
    4. Touch events

  • preventDefault() is a method of the event object that can be used inside a callback function to stop a default behavior from happening

  • Event propagation is the order that events 'fire' on each element.
  • There are 2 forms of event propagation: Bubbling and Capturing
  • Bubbling: when an event fires on the element clicked on first, then bubbles up the document tree, firing an event on each parent element until it reaches the root node
  • Capturing: starts by firing an event on the root element, then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on
  • Bubble phase can be stopped from occurring by adding the event.stopPropagation() method into the callback function

Event Delegation

  • Event delegation can be used to attach an event listener to a parent element in order to capture events that are triggered by its child elements
